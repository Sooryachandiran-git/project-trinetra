# Attack Modules

This directory contains cyber-physical attack simulations used by Project TRINETRA for isolated, authorized lab research.

## Purpose

The files here model different attack classes against the digital-twin environment so the orchestrator, PLC logic, telemetry path, and frontend can be studied under fault and adversarial conditions.

## Safety Note

These modules are security-sensitive. Do not use them against real devices, third-party systems, or any environment you do not own and explicitly control.

Execution instructions for offensive modules are intentionally not documented here. If you need help validating the project safely, use the backend and frontend startup flow in [../../milestones/execution_guide.md](../../milestones/execution_guide.md) and keep testing confined to an isolated local lab.
